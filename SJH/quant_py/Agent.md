# 총괄 Agent (Agent.md)

이 파일은 **퀀트투자 포트폴리오 웹 애플리케이션** 프로젝트의 전체적인 구조, 역할 정의 및 코어 비즈니스 로직을 총괄 기재하는 문서입니다.

---

## 📌 프로젝트 개요
- **목적**: 국내/해외 주식 데이터를 기반으로 퀀트 투자 전략(퀄리티, 밸류, 모멘텀 등)을 적용하여 포트폴리오를 선정하고 시뮬레이션할 수 있는 인터랙티브 웹 프로그램 구축.
- **주요 프레임워크**: Python + Streamlit + MySQL
- **에이전트 역할 분담**:
  1. **총괄 Agent (`Agent.md`)**: 전체적인 기획, 요구사항 정의, 화면 설계 및 기능적 구조 정의.
  2. **디자이너 Agent (`style.md`)**: 밝고 상큼한 디자인 톤앤매너, 컬러 가이드라인, UI/UX 디테일 정의.
  3. **개발자 Agent (`dev.md`)**: 시스템 아키텍처, 데이터베이스 스키마 및 파이프라인, 세부 구현 기술 정의.

---

## ⚙️ 주요 기능 요구사항

### 1. 퀀트 지표 필터링 & 종목 선정 (필수 기능)
- **퀄리티(Quality) 지표**: 
  - 자기자본이익률(ROE) ↑, 매출총이익(GPA) ↑, 영업활동현금흐름(CFO) ↑
  - 세 지표를 표준화(Z-score)하여 퀄리티 종합 점수(`z_quality`) 산출.
- **밸류(Value) 지표**: 
  - PER ↓, PBR ↓, PSR ↓, PCR ↓, DY(배당수익률) ↑
  - 지표들을 표준화하여 밸류 종합 점수(`z_value`) 산출.
- **모멘텀(Momentum) 지표**:
  - 3개월, 6개월, 12개월 수익률 및 K-Ratio 수치 제공.
  - 12개월 수익률 ↑, K-Ratio ↑ 요소를 기반으로 모멘텀 종합 점수(`z_momentum`) 산출.
- **종합 점수(QVM) 산출**: 
  - 퀄리티, 밸류, 모멘텀 점수를 합산(혹은 가중 합산)하여 최종 투자 대상(`invest = 'Y'`) 선정.
  - 연도별/최근4분기 기준 상위 20개 종목 추출 기능 제공.

### 2. 추천 추가 기능
- **포트폴리오 비중 최적화**: `riskfolio-lib`을 활용하여 최대 샤프지수(Max Sharpe), 최소 분산(Min Variance) 등 최적 비중 산출 및 도표화.
- **백테스팅 시뮬레이션**: 선정된 포트폴리오의 과거 누적 수익률, MDD(최대 낙폭), 샤프 지수를 벤치마크(KOSPI/KOSDAQ)와 비교하는 백테스트 시각화 화면.
- **실시간 주가 조회 및 개별 종목 차트**: 선정된 종목의 최근 주가 흐름을 인터랙티브 라인 차트(Plotly 등)로 제공.
- **데이터 업데이트 컨트롤 탭**: 데이터 수집 스크립트(`1.KOR_TICKER&SECTOR.py` ~ `5.kor_fs_value.py`)를 Streamlit 화면에서 실행하고 로그를 볼 수 있는 관리 기능.

## 🔄 프로젝트 수행과정 및 절차

본 프로젝트의 개발 및 운영 시 적용되는 전체적인 수행과정 및 절차는 다음과 같습니다.

### 1단계: 환경 설정 및 의존성 설치
- **작업 내용**: 파이썬 가상환경 구성, 필요한 패키지(`requirements.txt`에 명시된 `streamlit`, `pandas`, `riskfolio-lib` 등) 설치.
- **데이터베이스 구축**: 로컬 MySQL 서버에 `stock_db` 데이터베이스 및 관련 테이블(`kor_ticker`, `global_ticker` 등) 생성.
- **환경 변수 설정**: 데이터베이스 접속 정보와 패스워드(`1234`)를 `.env` 파일에 기록하여 보안 유지.

### 2단계: 주식 기초 데이터 수집 (파이썬 스크립트 실행)
- **수집 파이프라인**: 
  1. `1.KOR_TICKER&SECTOR.py` 실행 (시장별 티커 및 WICS 업종 분류 정보 적재)
  2. `2.KOR_PRICE.py` 실행 (종목별 1개년 주가 및 거래량 데이터 수재)
  3. `3.KOR_FS&VALUE.py` 실행 (재무제표 크롤링 및 주요 팩터 계산)
  4. `4.KOR_VALUE(공시구분 y).py` 실행 (연도별 리밸런싱용 지표 데이터 정합)
  5. `5.kor_fs_value.py` 실행 (전체 데이터 병합 및 `kor_multi_factor` 최종 테이블 적재)
- **글로벌 데이터 적재**: 해외 시장 조회를 위해 `sample_generator.py`를 실행하여 10대 US 종목 데이터를 모의 적재.
- *주*: 이 전체 과정은 Streamlit 화면의 **데이터 관리 및 업데이트** 탭에서 실시간 로그 확인과 함께 원클릭으로 실행 가능.

### 3단계: 멀티팩터(QVM) 스크리닝 및 실시간 연산
- **작업 내용**: 사용자가 웹 화면에서 퀄리티, 밸류, 모멘텀의 가중치를 직접 조절.
- **연산 절차**: 
  - 백엔드 모듈(`quant_helper.py`)에서 최신일자 데이터를 로드.
  - 아웃라이어 조율(1% 윈저라이징 클램핑) 수행.
  - 각 섹터별(Sector-Neutral)로 가중치에 따른 순위 Z-Score 계산.
  - 종합 점수(QVM)가 높은 상위 20개 종목을 실시간 정렬 및 도출.
  - 도출된 포트폴리오는 세션 상태(`st.session_state`)로 저장되어 다른 페이지와 연계.

### 4단계: 포트폴리오 비중 최적화 및 리밸런싱 백테스트
- **비중 최적화**: 
  - 선정된 20개 종목에 대해 `riskfolio-lib`을 호출.
  - 일별 수익률의 공분산 행렬을 추정하여 최대 샤프지수(Max Sharpe Ratio) 또는 최소 위험(Min Variance) 비중 산출 및 도넛 차트 출력.
- **백테스팅 시뮬레이션**:
  - 리밸런싱 주기(단순 보유, 매월, 분기별) 및 슬리피지/세금 비용률(예: 0.25%)을 입력.
  - 매 리밸런싱 시점마다 자산의 실제 비중과 목표 비중 차이를 비교하여 거래 비용 차감 후 복리 수익률 재연산 및 벤치마크(전체 동일비중)와 차트 비교.

### 5단계: 주문 신호 출력 및 가상 매매 연동
- **작업 내용**: 최종 포트폴리오 비중에 맞춰 실제 주문 수량(Shares) 연산.
- **연산 절차**:
  - 설정한 투자 원금(예: 1,000만 원)에 최적 비중을 곱하여 목표 금액 산출.
  - 현재 보유 수량과 비교하여 추가 매수/매도 수량을 정수로 산출.
  - 최종 주문 데이터프레임을 생성하고 Excel 호환 CSV 파일로 내보내기 다운로드 수행.

## 💡 README.md 및 교재(./html) 기반 개선점 및 기능 제안

책의 각 장(Html 문서)을 분석하여 Citrus Quant 웹 프로그램에 추가 적용 가능한 개선 사양을 제안합니다.

1. **전 세계 주식 데이터 수집 및 해외 포트폴리오 지원 (Ch 11)**
   - **설명**: 현재 국내 주식에 한정된 데이터를 해외(미국 등) 주식까지 확장합니다.
   - **역할**: `yfinance` 및 `yahooquery` 패키지를 연동하여 글로벌 시가총액 상위 종목의 QVM 스크리닝 및 포트폴리오 조성을 추가 지원합니다.

2. **기술적 지표 분석 차트 제공 (Ch 15)**
   - **설명**: 단순 주가 라인 차트를 넘어 투자자 분석을 고도화합니다.
   - **역할**: 개별 종목 조회 탭에서 이동평균선(MA), 볼린저 밴드(Bollinger Bands), RSI, MACD 등 트레이딩 결정을 돕는 보조지표 오버레이 차트를 Plotly로 제공합니다.

3. **주기적 리밸런싱 백테스팅 시뮬레이션 (Ch 16)**
   - **설명**: 단순 보유(Buy & Hold)가 아닌 현실적인 백테스트를 수행합니다.
   - **역할**: 매월 또는 분기별 리밸런싱을 수행하고 거래 수수료 및 슬리피지(Slippage) 비율을 대입할 수 있는 리밸런싱 엔진을 추가합니다.

4. **증권사 API 연동 가상 주문 및 신호 시트 (Ch 17)**
   - **설명**: 웹 프로그램에서 도출된 포트폴리오를 실제 거래로 연결합니다.
   - **역할**: 한국투자증권 등 국내 증권사 API 규격에 맞춰 선정된 종목의 가상 모의 주문을 전송하거나 주문용 CSV 템플릿을 다운로드하는 기능을 추가합니다.
