# 워크스루: 시트러스 퀀트 포트폴리오 웹 애플리케이션 최종 론칭

에이전트 역할 파일(`Agent.md`, `style.md`, `dev.md`)에 정의된 설계 기준을 바탕으로 **시트러스 퀀트(Citrus Quant)** 웹 프로그램을 고도화하고, 데이터 수집 파이프라인의 오류 수정 및 브라우저 수동 검증을 성공적으로 마쳤습니다.

---

## 🚀 주요 구현 및 개선 기능

### 1. 전 세계 주식 데이터 지원 (11장 연계)
- 미국 시장의 시가총액 상위 10대 테크 기업 데이터를 포함하는 별도의 글로벌 테이블군(`global_*`)을 구성했습니다.
- **시장 선택 토글**을 추가하여 국내 주식(KOSPI)과 미국 주식(US) 시장을 즉시 전환하며 분석할 수 있습니다.

### 2. 캔들스틱 및 기술적 보조지표 차트 오버레이 (15장 연계)
- 개별 종목 상세 조회 시 Plotly 기반의 인터랙티브 **캔들스틱 차트**가 렌더링됩니다.
- 주가 선에 **20일 이동평균선(SMA)**, 반투명 민트색 영역으로 채워진 **볼린저 밴드(Bollinger Bands)**, 하단 **14일 RSI 보조지표**와 교차(Crossover) 시점의 매수/매도 시그널 마커를 표시합니다.

### 3. 주기적 리밸런싱 백테스터 구현 (16장 연계)
- 단순 보유(Buy & Hold) 방식 외에 **매월(21영업일)** 및 **분기별(63영업일)** 포트폴리오 리밸런싱 루프를 연산합니다.
- 리밸런싱 시마다 설정한 거래 수수료 및 슬리피지 비율을 차감하여 복리 수익 흐름을 정교하게 시뮬레이션합니다.

### 4. 증권사 API 연동 가상 주문 신호 생성 (17장 연계)
- 사용자가 설정한 총 투자 자금에 가중치를 대입하여 필요한 주문 주수를 실시간 연산합니다.
- 한글 인코딩 깨짐을 방지하는 UTF-8-SIG 포맷의 **증권사 API 전송용 주문 파일(CSV) 다운로드** 기능을 제공합니다.

## 🔧 이번 수동 실행 및 트러블슈팅 내역

1. **KRX 중복 로그인 및 모달 우회 성공**
   - Selenium을 이용한 로그인 과정에서 "이미 로그인된 계정입니다." 팝업이 부모창이 아닌 `COMS001_FRAME` iframe 내부에 존재하는 것을 HTML 구조 분석을 통해 감지하였습니다.
   - 이에 맞추어 컨텍스트를 iframe에 머무르게 한 뒤 `btn-confirm` 버튼을 클릭하게 수정하여 **세션 쿠키 추출을 완벽히 우회 및 자동화**했습니다.
2. **`lxml` 파서 라이브러리 설치를 통한 크롤러 정상화**
   - FnGuide 재무 데이터 파싱(`pd.read_html`) 시 가상환경에 `lxml` 패키지가 누락되어 모든 수집 루프에서 오류가 발생했었습니다. `pip install lxml`을 실행해 종목별 재무 정보 수집이 에러 없이 돌 수 있도록 완벽하게 복구했습니다.
3. **`kor_fs_value` 누락 테이블 DDL 생성**
   - 5단계 스크립트 실행 대상인 `kor_fs_value` 테이블이 데이터베이스에 존재하지 않아 발생한 오류를 테이블 생성 DDL 스크립트를 작성하여 해결하고 데이터 적재 기반을 구축했습니다.
4. **QVM 산점도 Z-Score 음수 크기 에러(Plotly Size Property ValueError) 해결**
   - 팩터 Z-Score 점수가 음수가 될 때 Plotly scatter plot의 `size` 인자로 대입되면 오류가 나던 문제를, 최소값을 오프셋하여 크기를 강제로 양수로 변환하는 보정 컬럼(`qvm_size`)을 추가하여 에러를 말끔히 해결했습니다.
5. **수집 파이프라인 속도 단축**
   - 각 스크립트의 불필요한 `time.sleep(0.5)` 대기 시간을 `0.1`로 튜닝하여, 약 23분 가량 소요되던 크롤링 실행 시간을 약 4.6분(5배 단축)으로 단축시켰고 백그라운드 태스크로 연이어 런칭시켰습니다.
